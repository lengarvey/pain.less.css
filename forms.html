Forms

Here at pain.less.css we strive to achive the higher levels of customers satisfaction. And in order to do so, we don't add classes to the inputs.

Input fields

Input fields should just start looking good. No extra nothing required

Forms and Labels

Forms can be done with a generic structure of the FORM, P and LABEL/INPUT tags. 

<form>
  <p>
    <label>Username</label>
    <input type="text" placeholder="Nikolay">
  </p>
  <p>
    <label>Comment</label>
    <textarea placeholder="Your comment"></textarea>
  </p>
</form>

If you need to use a custom structure for your input fields or use them outside of a form, please use the .controls css class on the grouping element. 

<div class="controls">
  <label>Username</label>
  <input type="text" value="Nikolay">
</div>

Horizontal Forms

To make your forms horizontal either add the .horizontal class to your FORM tag or inject the class into the forms with LESS as a mix in. 

<form class="horizontal">
  <p>
    <label>Username</label>
    <input type="text" value="nikolay">
  </p>
  <p>
    <label>Password</label>
    <input type="password" value="secret">
  </p>
  <p>
    <label class="checkbox">
      <input type="checkbox">
      Remember me
    </label>
  </p>
</form>

Inline Forms

To make and inline form, please keep the same form structure as before and just add the .inline class to the FORM element. 

<form class="inline">
  <p>
    <input type="text" placeholder="Username">
  </p>
  <p>
    <input type="password" placeholder="Password">
  </p>
  <p class="buttons">
    <button type="submit">Log In</button>
  </p>
</form>

You also can use the .inline class in conjunction with the standard .group class to create forms where the submit buttons look like addons. 

<form class="inline group">
  <p>
    <input type="search" placeholder="Search">
  </p>
  <p class="buttons">
    <input type="submit" value="Go">
  </p>
</form>

You can have button sections on the both sides of the form if you need to. Just remember to mark them with the .buttons css class. 

<form class="inline group">
  <p class="buttons">
    <input type="reset" class="primary" value="Rest">
  </p>
  <p>
    <input type="search" placeholder="Search">
  </p>
  <p class="buttons">
    <input type="submit" value="Go">
  </p>
</form>

Required and Modified Fields

To make a block with an input field as required or failed/successed please use the .required or .success, .danger classes on an input-block section, for example. 

<form>
  <p class="required success">
    <label>Username</label>
    <input type="text" value="nikolay">
  </p>

  <p class="required danger">
    <label>Password</label>
    <input type="password" value="secret">
  </p>

  <p class="required info">
    <label>
      <input type="checkbox" />
      Remember me
    </label>
  </p>
</form>
