Layouts Basics

To achieve the best results and ensure your page flexibility for the future and good support across different devices, please use the following basic layout structure. 

<body>
  <header>
    <div class="paddings">
      Header content
    </div>
  </header>
  <main>
    <div class="paddings">
      <article>
        Main text content
      </article>
      <aside>
        Sidebar text
      </aside>
    </div>
  </main>
  <footer>
    <div class="paddings">
      Footer content
    </div>
  </footer>
</body>

Once you have the structure, then you can inject your one the layouts style in the BODY tag in your application.less file. 

body {
  .fixed-layout; // if you want a fixed-width layout
}

There are couple of layout styles available

  • .fixed-layout - the fixed width layout
  • .basic-layout - the basic full page width layout

Sidebars Position

Sidebars are done by using the ASIDE tag and position of the sidebar depends on where it's placed in the HTML structure. 

<main>
  <div class="paddings">
    <article>
      Main text content
    </article>
    <aside>
      Sidebar is on the right
    </aside>
  </div>
</main>

If you want your sidebar to be on the left side, just move it before the ARTICLE section. 

<main>
  <div class="paddings">
    <aside>
      Sidebar is on the left
    </aside>
    <article>
      Main text content
    </article>
  </div>
</main>

Another way, you can change the flex-box order for the ASIDE tag

You also can have sidebars on the both sides.

Grid System

The grid system in pain.less.css is dead simple, add the .grid class to an element and that's it. All the child elements will be considered grid cells, and you can change their size by adding the .half, .third or .quoter css classes to them.
